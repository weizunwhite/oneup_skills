# Paper Examples by Grade Level

## 小学低年级 Example

### Project: 自动浇花器

**摘要**
```
我家阳台上种了很多花，但是每次出去旅游的时候，花就没人浇水，回来的时候好几盆花都枯萎了。于是我用Arduino控制板、土壤湿度传感器和小水泵，做了一个自动浇花器。当土壤湿度低于30%的时候，水泵就会自动启动浇水，浇到湿度达到60%就停止。我测试了7天，花盆的土壤一直保持湿润，一盆花也没有枯萎。这个装置还可以用在学校的花坛里。
```

**课题由来 Example（自然体现学生独立发现问题）**
```
我家阳台上种了好多花，有绿萝、多肉，还有一盆茉莉花。去年暑假，我们全家去奶奶家住了三天。回来以后，我发现茉莉花的叶子都耷拉下来了，土摸上去干干的。妈妈说是因为三天没浇水。我觉得很可惜，就想：能不能做一个东西，让它自己浇水呢？
我问了爸爸，他说有一种传感器可以测量土壤里的水分。我觉得这个办法可以试一试。
```

**遇到困难 Example（调试细节体现真实参与）**
```
第一次测试的时候，水泵接上电以后一直不出水。我检查了好几遍线路，都没有发现问题。
后来老师帮我看了一下，发现水管里有空气，水抽不上来。老师教我把水管的一头放到水里，
用手捏几下把空气排出来，水泵就开始出水了。
第二个问题是传感器的数值一直跳来跳去。老师说可以多测几次取中间的值。我在程序里加了
这个功能以后，数值就稳定多了。
```

**心得体会**
```
通过做这个自动浇花器，我学到了很多东西。我知道了土壤湿度传感器可以测量土壤里的水分，还学会了怎么用面包板连接电路。最让我开心的是，我再也不用担心出门的时候花会枯萎了。在制作过程中，我遇到了水泵不出水的问题，后来发现是水管里有空气，把水管放到水里排掉空气就好了。这让我明白了遇到问题不要着急，要仔细观察找原因。以后我还想给浇花器加一个小屏幕，能显示现在的湿度是多少。
```

## 小学高年级 Example

### Project: 智能提醒喝水杯垫

**摘要**
```
很多小学生在学校经常忘记喝水，导致饮水量不足，影响身体健康。我通过问卷调查了班级里36位同学，发现有25位（69%）每天在学校的饮水量不到推荐标准的一半。针对这个问题，我设计了一个智能提醒喝水杯垫，使用Arduino Nano控制板、压力传感器和LED灯带。杯垫通过压力传感器检测水杯重量变化来判断喝水情况，如果超过40分钟没有喝水，LED灯带就会亮起提醒。经过连续5天的测试，使用杯垫的5位同学平均每天多喝了2杯水（约400ml），饮水量提升了45%。本项目的主要创新点是通过重量变化检测喝水行为，不需要改造水杯本身。
```

**发明缘由 Example（第一人称自然叙事，带数据）**
```
我注意到班上很多同学上午第三节课的时候就开始喊渴，但是他们的水杯里还有很多水，
只是忘记喝了。为了了解这个问题有多普遍，我设计了一份问卷，在班级里发放了36份。
调查结果显示，有25位同学（69%）表示经常忘记喝水，每天在学校的饮水量不到推荐
标准的一半。我又在网上查了资料，发现市面上的智能水杯价格都在100元以上，而且需要
专门买新杯子，不太方便。我就想：能不能做一个不需要改造水杯的提醒装置呢？
```

**调试困难 Example（体现独立解决问题的过程）**
```
在调试压力传感器时，我发现空杯子放上去和装满水的杯子放上去，传感器读数的差距
很小，程序很难区分"喝了一口"和"没喝"。我先试着换了一个量程更小的传感器，
但还是不太稳定。后来我想到可以让程序记录每次重量变化的幅度，如果重量突然减少
超过50g，就说明喝了一口水。用这个方法以后，识别的准确率从60%提升到了90%。
```

**方案对比 Example**
```
我想到了三种方案来提醒喝水：

| 方案 | 做法 | 优点 | 缺点 | 是否采用 |
|------|------|------|------|----------|
| 方案一：智能杯盖 | 在杯盖上装传感器 | 直接检测 | 需要改造水杯，不同杯子不通用 | ❌ |
| 方案二：智能杯套 | 做一个带传感器的杯套 | 可以套在不同杯子上 | 杯套容易滑落，不稳定 | ❌ |
| 方案三：智能杯垫 | 做一个带传感器的杯垫 | 不需要改造杯子，放上去就能用 | 只能在桌上使用 | ✅ |

经过分析，我选择了方案三（智能杯垫），因为它最方便，任何水杯放上去都能用，不需要做任何改变。虽然只能在桌上使用，但同学们在学校大部分时间都坐在座位上，杯子就放在桌上，所以这个缺点影响不大。
```

**测试记录 Example**
```
我请了5位同学帮忙测试，每人用5天：

| 同学 | 测试前每天喝水量 | 使用杯垫后每天喝水量 | 变化 |
|------|-----------------|---------------------|------|
| 小明 | 约400ml | 约700ml | +300ml |
| 小红 | 约500ml | 约800ml | +300ml |
| 小华 | 约300ml | 约800ml | +500ml |
| 小丽 | 约600ml | 约900ml | +300ml |
| 小刚 | 约350ml | 约800ml | +450ml |

平均每天多喝了约370ml的水。5位同学都说灯光提醒很有用，不会像闹钟那样吵到别人。
```

## 初中 Example

### Project: 基于UWB测距的便携式视力测量仪

**摘要**
```
传统视力检测依赖固定灯箱式视力表和人工卷尺测量5米标准距离，存在设备体积大、测距误差大（人工测量误差±20cm）、视标对比度受环境光影响等问题。本项目设计了一套基于UWB超宽带测距技术的便携式视力测量仪，采用"主控端+显示端"双端分离式架构。主控端使用ESP32-S3搭配DW1000 UWB模块实现自动精确测距（精度±10cm），显示端使用4.2寸电子墨水屏显示E字视标（对比度≥85%），两端通过UWB无线通信实现测距和指令传输。经测试，系统测距成功率达96%，视力检测范围覆盖0.1至1.2，单次测试时间约3分钟，续航超过8小时，整机单端重量不足300g。主要创新点包括：UWB精确测距替代传统卷尺（精度提升50%）、电子墨水屏替代纸质/LCD视力表（对比度提升40%）、灯带颜色编码实现直觉化距离引导。本系统适用于学校、社区、户外等多场景视力筛查。

关键词：UWB测距；便携式视力检测；电子墨水屏；ESP32-S3
```

**调试过程 Example（专业语言，技术细节体现真实参与）**
```
8.2 调试中遇到的问题与解决

问题1：UWB测距数据波动。初始测试中，DW1000模块在5米距离上的测距结果波动范围
达±30cm，远超设计目标（±10cm）。经分析，波动主要来源于多径反射效应——测试环境
（教室）内的金属课桌和墙壁对UWB信号产生了反射干扰。通过在接收端增加10次采样的
中值滤波算法，将波动范围压缩至±10cm以内。

问题2：电子墨水屏刷新延迟。E-ink屏全刷新需要约2秒，导致切换视标时出现明显等待。
经过查阅GxEPD2库的文档，发现可以使用局部刷新模式（partial refresh），将视标切换
时间从2秒缩短至0.3秒。但局部刷新在连续使用15次后会出现残影，因此在程序中设置了
每15次局部刷新后执行一次全刷新的策略。

问题3：两端通信丢包。主控端和显示端通过UWB通道传输指令时，初始丢包率约为8%。
排查发现是发送间隔设置过短（10ms），两端的DW1000模块来不及完成收发切换。将发送
间隔调整至50ms后，丢包率降至1%以下。
```

**创新点 Example**
```
本项目的创新点包括：

1. **UWB精确测距替代传统卷尺**
   - 传统方案：人工卷尺测量，误差±20cm，需要额外人手
   - 本项目方案：DW1000 UWB模块自动测距，精度±10cm，无需人工操作
   - 优势：测距精度提升50%，同时省去一名测距人员

2. **电子墨水屏替代纸质/LCD视力表**
   - 传统方案：纸质视力表易污损，LCD屏阳光下看不清
   - 本项目方案：4.2寸电子墨水屏，对比度≥85%，阳光下清晰可视
   - 优势：对比度提升约40%，零功耗保持显示，户外可用

3. **灯带颜色编码实现直觉化距离引导**
   - 传统方案：无距离实时反馈
   - 本项目方案：WS2812B灯带红/蓝/绿三色指示距离状态
   - 优势：操作者无需查看数字，通过颜色即可判断距离是否达标

4. **双端分离便携设计**
   - 传统方案：灯箱式一体机，重量5-10kg，需固定安装
   - 本项目方案：主控端+显示端各自独立，单端<300g
   - 优势：重量降低90%以上，可装入书包携带
```

**测试与分析 Example**

## 高中 Example

### Abstract Example
```
摘要：据国家卫健委2023年统计数据显示，我国儿童青少年近视率已达52.7%，其中小学生近视率为35.6%。现有视力检测设备存在体积大、便携性差、测距精度低等问题，难以满足基层学校和社区的大规模视力筛查需求。本研究提出一种基于UWB（Ultra-Wideband）超宽带技术的便携式视力测量系统，采用DW1000模块实现TWR（Two-Way Ranging）双向测距，经中值滤波后测距精度达±10cm（n=50，95% CI）。系统采用ESP32-S3双核处理器驱动4.2寸电子墨水屏显示标准E字视标（GB 11533-2011），视标对比度≥85%。经50名受试者（年龄8-15岁）双盲对照实验验证，本系统检测结果与标准灯箱视力表的一致性达到93.6%（κ=0.87），单次检测时间缩短至3分钟（传统方法约8分钟），整机单端重量不足300g。本研究为基层视力筛查提供了一种高精度、便携化的技术方案。

关键词：UWB超宽带测距；便携式视力检测；电子墨水屏；TWR双向测距算法；ESP32-S3
```

## Common Pitfalls by Grade

### 小学低
- ❌ 使用"本装置""本系统"等指代
- ❌ 列出复杂的技术参数表
- ❌ 使用学术引用格式
- ✅ 用"我做的浇花器"代替"本装置"
- ✅ 简单说"传感器能感觉到土壤干不干"

### 小学高
- ❌ 没有调研数据，直接开始做
- ❌ 方案选择没有理由，直接说"我选了方案三"
- ❌ 测试只说"成功了"没有数据
- ✅ 有问卷或访谈数据支撑问题的普遍性
- ✅ 方案对比有表格有理由
- ✅ 测试有具体次数和结果

### 初中
- ❌ 口语化表达："我觉得挺好的"
- ❌ 缺少技术细节：不说用什么芯片、什么通信协议
- ❌ 测试数据只有一轮
- ✅ 严谨的技术描述
- ✅ 多轮测试 + 失败分析
- ✅ 创新点有量化对比

### 高中
- ❌ 缺少文献综述
- ❌ 没有对照实验
- ❌ 数据没有统计分析
- ✅ 引用相关研究文献
- ✅ 设计对照实验
- ✅ 报告均值、标准差、置信区间
