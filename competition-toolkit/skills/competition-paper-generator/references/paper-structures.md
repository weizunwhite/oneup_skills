# Paper Structures by Grade Level

每个年级提供多种结构变体。所有变体覆盖相同的"内容原子"（摘要、背景、目标、设计、实现、调试、测试、创新点、局限、心得、致谢、参考文献），只是组织方式不同。

---

## 小学低年级（1-3年级）

**字数**: ~3000字
**语言**: 口语化，第一人称"我"，自然的儿童表达
**重点**: 观察力、好奇心、动手尝试的真实过程

### Variant A "故事型"（默认）

像在讲一个完整的小故事，有起因、经过、结果。

```
1. 摘要（100-150字）
2. 我发现了什么问题
3. 我是怎么想到解决办法的
4. 我用了什么材料
5. 我是怎么做的（分步骤）
6. 做出来的效果
7. 还可以怎么改进
8. 心得体会
9. 致谢
```

### Variant B "问答型"

以孩子自然提出的问题串起全文，每章回答一个问题。

```
1. 摘要（100-150字）
2. 为什么要做这个？（发现问题、生活观察）
3. 想做成什么样的？（目标、初步想法）
4. 怎么做出来的？（材料+分步骤制作过程）
5. 遇到什么困难？怎么解决的？（调试、失败、改进）
6. 好不好用？（测试效果、让别人试用）
7. 以后还能怎么改？（不足与改进）
8. 心得体会
9. 致谢
```

### Variant C "日记型"

用日记式的时间推进叙事，像在翻阅一本小小的项目日记。

```
1. 摘要（100-150字）
2. 故事的开始（发现问题、产生想法）
3. 我的计划（设计方案、准备材料）
4. 动手制作（分步骤做、遇到困难）
5. 试一试（测试效果、找人试用）
6. 还能更好（不足与改进想法）
7. 心得体会
8. 致谢
```

### 各章节通用说明

以下说明适用于所有小学低年级变体，不论哪种结构都需要包含这些内容要素：

**摘要（100-150字）**
用最简单的语言概括：发现了什么问题、做了什么东西、效果怎么样。
要有1-2个具体的效果描述（如"可以在10秒内完成浇水"）。

**发现问题/背景（300-400字）**
- 从真实的个人经历出发
- 描述观察到的现象
- 可以问身边的人，简单调查

**想办法/设计（200-300字）**
- 思考过程
- 查了什么资料
- 和谁讨论过
- 最初的想法是什么

**材料（200-300字）**
- 列出所有用到的材料和工具
- 可以用简单表格
- 说明在哪里买的/找的

**制作过程（600-800字）**
- 分步骤叙述，每步配图
- 说明遇到的困难和怎么解决的
- "第一步...第二步..."的格式

**测试效果（300-400字）**
- 描述作品的功能
- 简单的测试（试了几次，成功了几次）
- 让别人试用的反馈

**改进（150-200字）**
- 自己觉得哪里不够好
- 如果再做一次会怎么改

**心得体会（200-300字）**
- 学到了什么知识
- 学到了什么技能
- 有什么感受
- 对以后有什么启发

**致谢（100-150字）**
- 感谢指导老师
- 感谢爸爸妈妈的帮助
- 感谢学校提供的条件
- 感谢帮助测试的同学

## 小学高年级（4-6年级）

**字数**: ~5000字
**语言**: 自然的学生表达，可以口语化，但有条理
**重点**: 完整的发明过程，有调研数据支撑

### Variant A "发明报告型"（默认）

经典的发明过程记录，从背景到测试，层层推进。

```
1. 摘要（200字左右）
2. 发明背景
   2.1 发明缘由（个人经历）
   2.2 调查研究（问卷/访谈数据）
   2.3 现有产品调研
3. 设计方案
   3.1 设计目标
   3.2 方案对比与选择
   3.3 功能设计
   3.4 材料清单
4. 制作过程
   4.1 制作步骤（分步图文）
   4.2 遇到的困难与解决
   4.3 调试与改进
5. 测试记录
   5.1 功能测试
   5.2 实用性测试（找人试用）
6. 创新点（3-4个）
7. 不足与改进方向
8. 心得体会
9. 致谢
10. 参考资料
```

### Variant B "探究型"

以科学探究的思路组织，强调猜想-验证的过程。

```
1. 摘要（200字左右）
2. 我的疑问（发现问题、提出疑问、调查数据）
3. 我的猜想（假设、初步思路）
4. 我的方案（设计目标、方案对比选择、材料准备）
5. 动手验证（制作步骤、遇到困难、调试改进）
6. 结果分析（功能测试数据、试用反馈、与猜想对比）
7. 我的发现（创新点总结）
8. 还可以做什么（不足与改进方向）
9. 心得体会
10. 致谢
11. 参考资料
```

### Variant C "设计思维型"

用设计思维（Design Thinking）的流程组织，强调用户需求驱动。

```
1. 摘要（200字左右）
2. 了解需求（发现问题、调研用户需求、问卷/访谈数据）
3. 头脑风暴（多种方案构想、方案对比与选择）
4. 设计原型（设计目标、功能设计、材料清单）
5. 制作与改进（制作步骤、遇到困难、调试改进）
6. 测试与反馈（功能测试、找人试用、收集反馈）
7. 创新总结（创新点）
8. 未来计划（不足与改进方向）
9. 心得体会
10. 致谢
11. 参考资料
```

### 各章节通用说明

以下说明适用于所有小学高年级变体：

**摘要（200字）**
必须包含：问题是什么→用什么方法解决→核心技术→具体效果数据→主要创新点。
不说"效果良好"，要说具体数据。

**背景/问题（600-800字）**
- 从真实经历引出问题
- 设计问卷/访谈，用图表展示数据（如"调查了30位同学，其中25位表示..."）
- 调研市面上的产品，列表对比优缺点

**设计方案（600-800字）**
- 列出3-4个具体目标
- 设计2-3种方案，用表格对比，说明选择理由
- 画功能框图或列出功能清单
- 材料清单表格（名称、数量、用途、图片）

**制作过程（800-1000字）**
- 分步骤，每步有文字说明和配图位置标注
- 至少描述2-3个遇到的困难，以及如何解决
- 调试过程中的数据记录（如传感器校准数值表格）

**测试记录（500-600字）**
- 功能测试：多次测试的数据表格
- 实用性测试：找3-5位不同的人试用，记录反馈

**创新点（300-400字）**
- 列出3-4个创新点
- 每个简要说明"现有方案怎样→我的方案怎样→好在哪"

**不足与改进方向（200-300字）**
- 诚实列出2-3个不足
- 提出具体改进思路

**心得体会（300-400字）**
- 技术能力的提升
- 解决问题能力的提升
- 团队协作/自主学习的体验
- 对科技创新的理解

**致谢（100-200字）**

**参考资料（5条以上）**

## 初中（7-9年级）

**字数**: ~8000-10000字
**语言**: 严谨正式，"我"与"本系统/本装置/本项目"混用，技术术语准确
**重点**: 系统性设计、技术深度、数据支撑

### Variant A "工程报告型"（默认）

硬件和软件分开成独立章节。适合有明确硬件/软件分离的传统嵌入式项目。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 项目背景
   2.1 问题现状分析
   2.2 现有解决方案对比
   2.3 问题提出
3. 研究目标（量化技术指标）
4. 系统设计
   4.1 总体架构（架构图）
   4.2 核心模块说明
   4.3 通信协议/数据流
5. 硬件设计
   5.1 元器件选型与理由
   5.2 硬件连接（引脚表）
   5.3 外观/结构设计
6. 软件设计
   6.1 程序流程（流程图）
   6.2 核心算法说明
   6.3 关键代码片段
7. 制作与调试
   7.1 制作过程
   7.2 调试中遇到的问题与解决
8. 测试与分析
   8.1 测试方案
   8.2 功能测试（数据表格）
   8.3 性能测试（多轮数据）
   8.4 对比分析（与现有方案）
   8.5 用户测试反馈
9. 创新点（量化对比）
10. 局限与改进方向
11. 心得体会
12. 致谢
13. 参考文献（8条以上）
```

### Variant B "问题驱动型"

围绕 2-3 个核心技术问题组织，每个问题自成一章（分析→设计→实现→验证）。适合项目有多个相对独立的技术难点。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 问题调研
   2.1 现状分析
   2.2 现有方案分析
   2.3 解决思路
3. 研究目标（量化技术指标）
4. 总体方案设计
   4.1 系统架构
   4.2 技术路线
5. 核心问题一：[如"药品识别"]
   5.1 问题分析
   5.2 方案设计
   5.3 实现过程
   5.4 单元验证
6. 核心问题二：[如"精确定位"]
   6.1 问题分析
   6.2 方案设计
   6.3 实现过程
   6.4 单元验证
7. 核心问题三：[如"流程闭环"]
   7.1 问题分析
   7.2 方案设计
   7.3 实现过程
   7.4 单元验证
8. 系统集成与联调
   8.1 集成过程
   8.2 联调问题与解决
   8.3 完整系统测试
9. 创新总结（量化对比）
10. 不足与展望
11. 心得体会
12. 致谢
13. 参考文献（8条以上）
```

### Variant C "迭代演进型"

按时间线的开发迭代组织，强调逐步改进过程。最能体现学生真实参与，适合有明显版本迭代或改进过程的项目。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 项目背景与动机
   2.1 问题发现
   2.2 调研分析
   2.3 研究目标
3. 方案调研与初步设计
   3.1 现有方案对比
   3.2 初始设计方案
   3.3 材料与工具准备
4. 第一轮实现：基础功能
   4.1 硬件搭建与软件开发
   4.2 初步测试
   4.3 发现的问题
5. 第二轮改进：核心优化
   5.1 问题分析与改进方案
   5.2 改进实现
   5.3 改进前后对比（量化数据）
6. 第三轮完善：系统完善
   6.1 补充功能开发
   6.2 外观与结构优化
   6.3 最终测试
7. 测试与评估
   7.1 完整功能测试（数据表格）
   7.2 与研究目标对比
   7.3 用户反馈
8. 创新点（量化对比）
9. 局限与未来改进
10. 心得体会
11. 致谢
12. 参考文献（8条以上）
```

### Variant D "功能模块型"

每个功能模块自成一章，每章包含该模块的需求、硬件、软件、调试。适合有 2-4 个可分离的设备/功能模块的项目。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 项目背景
   2.1 问题分析
   2.2 现有方案调研
   2.3 解决思路
3. 研究目标与系统概述
   3.1 研究目标（量化指标）
   3.2 系统架构
   3.3 技术路线
4. 模块一：[如"感知模块"]
   4.1 模块需求
   4.2 硬件设计（选型/连接）
   4.3 软件实现
   4.4 模块调试
5. 模块二：[如"执行模块"]
   5.1 模块需求
   5.2 硬件设计
   5.3 软件实现
   5.4 模块调试
6. 模块三：[如"交互模块"]
   6.1 模块需求
   6.2 硬件设计
   6.3 软件实现
   6.4 模块调试
7. 系统集成测试
   7.1 集成过程
   7.2 功能测试（数据表格）
   7.3 性能测试
   7.4 与研究目标对比
8. 创新点（量化对比）
9. 局限与改进
10. 心得体会
11. 致谢
12. 参考文献（8条以上）
```

### 各章节通用说明

以下说明适用于所有初中变体：

**摘要（300字）+ 关键词**
这是评委唯一完整阅读的部分，必须精炼有力：
- 背景问题（2句）
- 技术方案概述（2句，包含核心技术名称）
- 具体量化结果（2-3句，必须有数据）
- 创新点（1-2句）
- 应用价值（1句）
关键词：3-5个核心技术词

**项目背景（800-1000字）**
- 用数据说明问题的普遍性和严重性（引用统计、新闻、调研结果）
- 分析现有解决方案，用表格对比各方案的优缺点
- 指出现有方案的不足，引出本项目要解决的核心问题

**研究目标（300-400字）**
列出可量化的技术指标，例如：
- 测距精度：±Xcm
- 响应时间：<Xms
- 成功率：≥X%
- 续航时间：≥X小时
- 整机重量：<Xg（或体积：<X×X×Xcm）

**设计章节（800-1000字）**
- 总体架构图，说明各模块作用
- 每个核心模块的功能和选型理由表格
- 模块间的通信方式和数据流向

**硬件设计（600-800字）**
- 关键元器件选型表格（名称、型号、参数、选型理由）
- 引脚连接表（外设→连接引脚→通信方式）
- 外观结构设计说明

**软件设计（600-800字）**
- 程序流程图
- 核心算法的文字说明（原理+步骤）
- 关键代码片段（注释清晰）

**制作与调试（600-800字）**
- 关键制作步骤
- 至少3个调试问题及解决方案（体现真实性）

**测试与分析（800-1200字）**
- 测试环境和方案描述
- 功能测试数据表格（测试项→预期→实际→结论）
- 性能测试多轮数据（表格+分析）
- 与现有方案的量化对比表格
- 用户反馈统计

**创新点（400-500字）**
每个创新点用"现有方案→本项目方案→量化优势"的结构。

**局限与改进方向（300-400字）**
- 诚实列出3-4个局限
- 每个局限对应一个具体改进思路

**心得体会（400-500字）**
- 技术能力提升（具体技术）
- 工程思维的建立
- 问题解决能力的提升
- 科学态度和严谨性
- 对科技创新的理解

**致谢（150-200字）**

**参考文献（8条以上，GB/T 7714格式）**

---

## 高中（10-12年级）

**字数**: ~12000-15000字
**语言**: 学术化，"本研究""本文"，专业术语准确，引用规范
**重点**: 研究深度、学术规范、统计分析、创新性论证

### Variant A "学术论文型"（默认）

经典学术论文结构，引言含文献综述，设计章节分硬件/软件。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 引言
   2.1 研究背景
   2.2 文献综述
   2.3 研究意义
3. 研究目标与假设
4. 系统设计
   4.1 总体架构（架构图）
   4.2 技术方案论证
   4.3 通信协议设计
5. 硬件设计
   5.1 元器件选型论证
   5.2 电路设计
   5.3 结构设计
6. 软件设计
   6.1 软件架构
   6.2 核心算法（含伪代码/公式）
   6.3 关键实现
7. 制作与调试
   7.1 开发环境
   7.2 制作过程
   7.3 调试记录
8. 实验设计
   8.1 实验方案
   8.2 测试环境与条件
   8.3 对照实验设计
9. 结果与分析
    9.1 功能验证
    9.2 性能测试数据
    9.3 统计分析
    9.4 误差分析
    9.5 与现有方案对比
    9.6 用户评估
10. 创新点与贡献
11. 局限性讨论
12. 结论与展望
13. 心得体会
14. 致谢
15. 参考文献（10条以上）
16. 附录（可选）
```

### Variant B "研究驱动型"

以研究问题和假设为主线，设计与实现合并，强调实验验证和结果讨论。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 引言
   2.1 研究背景
   2.2 文献综述
   2.3 研究意义
3. 研究问题与假设
   3.1 研究问题
   3.2 研究假设
4. 研究方法
   4.1 技术路线
   4.2 系统设计
   4.3 实验设计
5. 系统实现
   5.1 硬件实现
   5.2 软件实现
   5.3 系统集成与调试
6. 实验过程
   6.1 实验环境与条件
   6.2 实验步骤
   6.3 数据采集方法
7. 结果与讨论
   7.1 实验结果
   7.2 数据分析（统计检验）
   7.3 假设验证
   7.4 与已有研究对比
8. 创新与贡献
9. 局限与未来工作
10. 结论
11. 心得体会
12. 致谢
13. 参考文献（10条以上）
14. 附录（可选）
```

### Variant C "工程研究型"

独立文献综述章，设计与实现合并为一章，评估方法独立成章。

```
1. 摘要（300字）+ 关键词（3-5个）
2. 引言
   2.1 研究背景
   2.2 研究意义
3. 相关工作
   3.1 国内外研究现状
   3.2 现有方案分析
   3.3 研究切入点
4. 需求分析与研究目标
   4.1 需求分析
   4.2 研究目标
5. 系统设计与实现
   5.1 总体架构
   5.2 模块一设计与实现
   5.3 模块二设计与实现
   5.4 模块三设计与实现
   5.5 系统集成
6. 评估方法
   6.1 评估指标
   6.2 实验环境
   6.3 对照设计
7. 实验结果与分析
   7.1 功能验证
   7.2 性能测试
   7.3 统计分析
   7.4 与现有方案对比
8. 讨论
   8.1 创新点
   8.2 局限性
   8.3 改进方向
9. 结论
10. 心得体会
11. 致谢
12. 参考文献（10条以上）
13. 附录（可选）
```

### 各章节通用说明

以下说明适用于所有高中变体：

**摘要（300字）+ 关键词**
学术摘要格式，但必须包含具体量化数据。

**引言（1200-1500字）**
- 研究背景：宏观到微观，引用权威数据
- 文献综述：分析5-8篇相关文献/产品，总结现状与不足
- 研究意义：理论意义+实践意义

**研究目标与假设（400-500字）**
- 明确量化目标
- 提出可验证的研究假设

**设计章节（2500-3500字）**

**制作与调试（800-1000字）**

**实验设计（600-800字）**

**结果与分析（1500-2000字）**

**创新点与贡献（500-600字）**
学术式创新点论证，引用文献对比。

**局限性讨论（400-500字）**
深入分析技术限制和改进可能。

**结论与展望（400-500字）**
总结研究发现，展望短期/中期/长期改进方向。

**心得体会（400-500字）**

**致谢（200字）**

**参考文献（10条以上，GB/T 7714格式）**

**附录**（完整代码、电路图、原始数据等）

## 研究方案（开题报告）结构

适用于所有年级，篇幅约为论文的1/3。

```
1. 项目名称
2. 项目背景（简述问题和意义）
3. 研究目标（量化指标）
4. 研究思路与技术路线
5. 创新点
6. 研究方法
7. 研究计划（时间表+里程碑）
8. 预期成果
9. 参考资料
```

小学低年级的研究方案可以更简单：项目名称→我要解决什么问题→我打算怎么做→需要什么材料→时间安排→希望做出什么效果。

## 自动选择算法

生成论文时，根据以下规则自动选择结构变体，**不需要与用户确认**。

### 初中 / 高中

按优先级依次检查，命中即停：

1. **分析原始资料中的独立技术问题数量**
   - 项目有 2-3 个各自独立的核心技术难点（如"药品识别"+"精确定位"+"流程闭环"）→ **Variant B（问题驱动型 / 研究驱动型）**

2. **检查是否有明显的迭代改进证据**
   - 技术方案中提到版本变更、重新设计、方案对比后更换、前后性能对比 → **Variant C（迭代演进型 / 工程研究型）**

3. **统计功能模块数量**（仅初中）
   - 项目有 2-4 个物理上可分离的设备或功能模块单元 → **Variant D（功能模块型）**

4. **以上都不突出** → **Variant A（工程报告型 / 学术论文型，默认）**

5. **平局时**：用 `hash(项目名称) % N` 决定（N = 匹配的变体数量）

### 小学（低年级 + 高年级）

所有变体都适用于所有小学项目，直接用轮转选择：

```
variantIndex = hash(项目名称) % 3
选择 Variant [A, B, C][variantIndex]
```

### Hash 函数定义

```
hash(s) = s 中每个字符的 charCode 之和
```

例如："医院自动取药系统" → 各字符 charCode 求和 → 取模选择变体。
