# Writing Guidelines

## Universal Rules (All Grades)

### Forbidden Patterns

**NEVER use metaphors, analogies, or personification for technical components (ALL grade levels):**
This is a hard rule regardless of the student's age. Even for 小学低年级, technical descriptions must be factual and direct.
- ❌ "ESP32就像大脑一样控制着整个系统"
- ❌ "摄像头是装置的眼睛"
- ❌ "传感器充当了电子鼻子的角色"
- ❌ "灯带就像一条彩虹"
- ❌ "CPU是整个系统的大脑"
- ❌ "超声波传感器就像蝙蝠一样发出信号"
- ❌ "电池为系统提供了生命力"
- ❌ "数据在模块之间流淌"
- ✅ "ESP32主控芯片负责数据采集和处理"
- ✅ "摄像头模块用于图像识别"
- ✅ "MQ-2烟雾传感器检测空气中的烟雾浓度"
- ✅ "WS2812B灯带显示不同颜色表示系统状态"
- ✅ "超声波传感器通过发射和接收声波来测量距离"

**Why this matters:** 比喻会让论文显得不专业、不严谨。科技论文的核心是准确描述技术事实，即使是低年级学生，也应该用简单但准确的语言，而不是文学化的类比。

**NEVER use vague effect descriptions in abstracts:**
- ❌ "取得了良好的效果"
- ❌ "性能得到了显著提升"
- ❌ "用户反馈很好"
- ✅ "测距精度达到±10cm"
- ✅ "检测准确率为97.4%"
- ✅ "95%的测试用户表示操作便捷"

**NEVER exaggerate or fabricate data.**

**NEVER use overly sophisticated language that doesn't match the student's grade level.**

### Required Sections

Every paper, regardless of grade, must include:

1. **心得体会** — near the end of the paper
2. **致谢** — final section before references

### Demonstrating Student Authenticity (替代独立「个人贡献」章节)

评委判断「是不是学生自己做的」靠的不是一段声明，而是全文的细节。不设独立的「个人贡献」章节，而是通过以下方式在全文中自然体现学生的独立参与：

**课题由来/发明背景/项目背景：**
用真实的个人经历引出问题。小低组自然地讲"我发现了..."；小高组描述"在日常生活中，我经常遇到..."；初中组用数据和调研引出，但背景仍要体现个人观察。

**制作过程/调试环节：**
这是最强的真实性证据。详细写遇到的具体困难、尝试的解决方法、失败后的调整——这些细节是成人代劳无法伪造的。至少包含2-3个真实的调试问题。

**收获与体会（官方标准章节）：**
总结个人技术成长、思维提升、项目感悟。这里自然包含了"我做了什么、学到了什么"。

**致谢：**
通过感谢具体的帮助内容（如"感谢张老师在UWB通信协议调试方面的指导"），间接说明了哪些是老师帮助的、哪些是自己完成的。

**集体项目的分工说明：** 如果是集体项目（2-3人），在研究方法或制作过程章节中自然提及各成员的分工即可。

### Abstract Writing Formula

Regardless of grade, every abstract follows this pattern (with age-appropriate language):

```
[问题背景 1-2句] + [技术方案 1-2句] + [量化结果 2-3句] + [创新亮点 1-2句] + [应用价值 1句]
```

The abstract should give an evaluator a complete picture of the project in under 30 seconds of reading.

## 小学低年级 Writing Style

### Language
- 完全口语化，第一人称"我"
- 可以出现"爸爸""妈妈""老师"等称呼
- 可以表达情感："我很开心""我有点担心"
- 句子短，结构简单
- 可以用"首先...然后...最后..."等连接词

### Tone
- 像在讲一个自己做实验的故事
- 充满好奇心和探索精神
- 遇到困难时诚实表达
- 成功时的真实喜悦

### Example Expressions
- "有一天，我发现..."
- "我想了很久，觉得可以..."
- "我试了好几次，第一次失败了，因为..."
- "后来爸爸建议我..."
- "最后终于成功了！"
- "通过这个项目，我学到了..."

### Avoid
- 任何学术化表达
- 复杂的技术术语（用简单描述代替）
- 长句子和复杂句式
- 被动语态

---

## 小学高年级 Writing Style

### Language
- 自然的学生表达，允许适度口语化
- 第一人称"我"为主
- 可以提到家人和老师的帮助
- 技术术语出现时给简单解释
- 有一定的条理性，但不要过于正式

### Tone
- 有条理地讲述发明过程
- 体现自主思考和解决问题
- 数据意识（会用表格和数字说明问题）
- 适当的反思和总结能力

### Example Expressions
- "在日常生活中，我经常遇到这样的问题..."
- "为了了解大家是否也有同样的困扰，我设计了一份问卷..."
- "经过对比分析，我最终选择了方案二，因为..."
- "在制作过程中，遇到了一个困难：...我尝试了...最终..."
- "测试结果显示..."
- "我认为还可以从以下方面改进..."

### Data Presentation
- 问卷结果用百分比和简单图表
- 测试数据用表格
- 对比用表格（方案A vs 方案B）
- 材料清单用表格

### Avoid
- 过于学术化的表达
- "本系统""本装置"等指代（用"我的作品""这个装置"代替）
- 复杂的统计分析
- 文献引用（可以简单提到"我查阅了资料发现..."）

## 初中 Writing Style

### Language
- 严谨正式，但不必完全学术化
- "我"和"本系统/本装置/本项目"混用
  - 描述个人行为时用"我"："我设计了..."
  - 描述系统特性时用"本系统"："本系统采用..."
- 技术术语直接使用，必要时在首次出现时给出简要解释
- 句式完整，逻辑连接词恰当

### Tone
- 工程师视角，注重逻辑和数据
- 客观描述技术方案和测试结果
- 遇到问题时用分析的方式说明原因和解决方法
- 创新点用对比数据支撑而非主观评价

### Example Expressions
- "通过调查研究发现，目前...存在以下问题..."
- "针对上述问题，本项目设计了一套..."
- "系统采用ESP32-S3作为主控芯片，通过SPI总线与DW1000 UWB模块通信"
- "在调试过程中，发现传感器读数存在±5%的波动，经分析是由于...，通过增加中值滤波算法解决了该问题"
- "测试结果表明，本系统在...指标上达到了设计要求"
- "与传统方案相比，本项目在体积上缩小了60%，同时精度提升了40%"

### Data Presentation
- 所有技术指标量化
- 多轮测试数据表格
- 对比分析表格（本项目 vs 现有方案）
- 硬件连接表、元器件参数表
- 用户测试统计

### Avoid
- 口语化表达："我觉得""我试了试"
- 模糊描述："效果很好""速度很快"
- 过度学术化：不需要文献综述
- 直接提到家长帮助（在致谢中提即可，正文中提"在老师指导下"）

## 高中 Writing Style

### Language
- 学术化，"本研究""本文""本系统"
- 被动语态和主动语态结合
- 专业术语规范使用
- 引用格式严格（GB/T 7714）

### Tone
- 研究者视角
- 注重理论依据和方法论
- 批判性思维（讨论局限性和改进空间）
- 文献驱动（现有研究 → 不足 → 本研究的切入点）

### Example Expressions
- "据统计数据显示，我国每年因...造成的损失达到..."
- "文献[3]提出了一种基于...的方法，但存在...的局限"
- "本研究提出一种基于...的解决方案"
- "采用TWR（Two-Way Ranging）双向测距算法，通过测量信号往返飞行时间计算距离"
- "实验结果表明，在95%置信区间内，系统测距误差为±10cm"
- "本研究的主要贡献包括：（1）...（2）...（3）..."

### Data Presentation
- 正式的实验方案（测试环境、条件、方法）
- 对照实验设计
- 统计分析（均值、标准差、置信区间）
- 误差分析
- 多维度对比表格

### Avoid
- 任何口语化表达
- 第一人称"我"（用"本文作者""本研究"替代）
- 情感化表达
- 非学术来源的引用

## Authenticity Through Writing — 通过写作细节体现学生真实参与

### Purpose
不通过独立声明，而是通过全文的叙事细节让评委自然感受到"这是学生自己做的"。

### 小学低年级 — 自然讲述，童真但准确
关键：像在跟老师讲自己做了一件什么事，自然、真实、不刻意。

**课题由来示例：**
```
有一天放学回家，我看到阳台上的花都蔫了。妈妈说是因为我们去奶奶家住了三天，没人浇水。
我就想：能不能做一个自动浇水的东西？
```

**遇到困难示例：**
```
我第一次接线的时候，水泵没有反应。我检查了好几遍，后来发现是正负极接反了。
换过来以后，水泵就转起来了。
```

**心得体会中自然带出个人成长：**
```
通过这个项目，我学会了用面包板接线，还学会了在Arduino里写简单的程序。
以前我觉得编程很难，现在发现一步一步来其实也能学会。
```

注意：小低组不使用比喻。"水泵负责把水从水箱抽到花盆里"✅，不说"水泵像心脏一样"❌

### 小学高年级 — 第一人称叙事，有数据支撑
关键：用"我"自然叙述，但体现出调研能力和动手过程。

**发明背景示例：**
```
在日常生活中，我经常看到爷爷忘记吃药。我问了班上30位同学，发现有18位同学
的家里也有老人忘记按时吃药的情况。我又在网上查了资料，发现市面上的智能药盒
价格都在200元以上，而且操作比较复杂，不太适合老年人使用。
```

**调试困难示例：**
```
在调试蜂鸣器提醒功能时，我发现蜂鸣器的声音太小，爷爷在客厅听不到。
我先试着换了一个更大的蜂鸣器，但功耗太大，电池撑不了一天。
后来我想到可以加一个小喇叭模块，只在提醒时才开启，这样既够响又省电。
```

### 初中 — 专业严谨，工程师视角
关键：用正式语言客观描述，通过技术细节的精确度和调试过程的深度来体现真实参与。不刻意声明"我做了什么"，而是让技术叙述本身说明一切。

**项目背景示例：**
```
通过调查研究发现，目前校园内的垃圾分类投放准确率不足60%。现有的智能垃圾桶
产品主要采用红外分类方案，识别准确率约为75%，且单台成本超过3000元，难以在
中小学大规模部署。针对上述问题，本项目设计了一套基于机器视觉的垃圾分类
辅助系统。
```

**调试过程示例（这是最能体现真实性的部分）：**
```
在调试图像识别模块时，初始模型对塑料瓶的识别准确率仅为62%。经分析发现，训练
数据集中塑料瓶的拍摄角度较为单一，导致模型对侧面和底部视角的识别能力不足。
将训练集从200张扩充至500张（增加了多角度、不同光照条件的样本）后，识别准确率
提升至89%。进一步调整置信度阈值从0.5至0.65，误识别率从12%降至4%。
```

**心得体会中的专业反思：**
```
通过本项目的开发，掌握了ESP32与MaixCAM之间的串口通信协议设计，理解了从需求分析
到系统测试的完整工程开发流程。在调试过程中，多次遇到硬件兼容性问题，逐步建立了
"先排查硬件、再排查软件、最后排查逻辑"的系统化调试思路。
```

### 高中 — 学术规范，研究者视角
关键：通过文献综述的深度、实验设计的严谨性、数据分析的完整性来体现研究能力。在"引言"中通过文献gap分析自然引出研究动机。

调试记录和实验方法的详细程度本身就是最强的真实性证据。

## 心得体会 Writing Guide

### Purpose
展示学生通过项目获得的素养和能力提升，这是评委评价教育价值的重要依据。

### 应涵盖的维度
1. **知识技能**：学到了什么具体技术或知识
2. **思维能力**：解决问题的方法、系统思维、批判性思考
3. **实践体验**：动手能力、调试耐心、失败中学习
4. **价值认识**：对科技创新的理解、对社会问题的关注
5. **未来启发**：对未来学习或项目的期待

### 小学低
侧重感受和简单的知识收获，用"我学到了..."的句式。

### 小学高
增加对过程的反思，可以讨论失败的经历和从中学到的东西。

### 初中
增加对工程思维和方法论的体会，可以讨论技术决策的过程。

### 高中
增加学术层面的反思，讨论研究方法的心得和学术态度的培养。

## 致谢 Writing Guide

### 致谢对象（按惯例顺序）
1. 指导老师（必须）
2. 父母/家人（必须）
3. 学校/实验室（提供场地、设备）
4. 帮助测试的同学/志愿者
5. 提供技术帮助的其他人

### 注意
- 真实、具体（"感谢张老师在电路调试方面的耐心指导"比"感谢老师的帮助"好）
- 篇幅适当，不过长
- 语气真诚

## 过渡语料库（Transition Corpus）

### 目的

不同项目使用不同的过渡表达，避免多个项目论文读起来像同一模板。

### 选择规则

用项目名称首字的 charCode 值决定使用哪套表达（套 0/1/2），同一项目内所有过渡语保持同一套。

```
transitionSet = projectName.charCodeAt(0) % 3  // 0, 1, 或 2
```

### 6 类过渡点 × 3 套表达

**1. 背景→问题引出**

| 套号 | 初中/高中 | 小学高 |
|------|-----------|--------|
| 0 | "通过调查研究发现，目前……存在以下问题" | "为了了解这个问题，我做了一些调查，发现……" |
| 1 | "深入调研后发现，现有……在实际应用中面临" | "我查了很多资料，还问了身边的人，发现……" |
| 2 | "经文献查阅和实地观察，当前……的主要不足在于" | "我观察了很长时间，又在网上搜索了相关信息，了解到……" |

**2. 设计章节过渡**

| 套号 | 初中/高中 | 小学高 |
|------|-----------|--------|
| 0 | "本系统采用……的总体架构" | "我决定用……来做这个装置" |
| 1 | "系统整体采用……架构，由……组成" | "经过比较，我选择了……的方案" |
| 2 | "在架构层面，采用……的设计思路" | "想了好几个方案之后，我觉得……最合适" |

**3. 测试章节过渡**

| 套号 | 初中/高中 | 小学高 |
|------|-----------|--------|
| 0 | "为验证系统性能，设计了以下测试方案" | "做好以后，我开始测试" |
| 1 | "测试分为X个维度，分别针对……" | "为了看看效果怎么样，我做了几组测试" |
| 2 | "按照研究目标中的各项指标，分别设计了对应的测试实验" | "接下来我请了几位同学帮忙试用" |

**4. 调试/迭代过渡**

| 套号 | 初中/高中 | 小学高 |
|------|-----------|--------|
| 0 | "在调试过程中，发现……" | "做的时候遇到了一个问题：……" |
| 1 | "实际测试时遇到了……的问题" | "第一次试的时候，发现……" |
| 2 | "系统运行中出现了……现象，经排查……" | "一开始没有成功，因为……" |

**5. 创新点过渡**

| 套号 | 初中/高中 | 小学高 |
|------|-----------|--------|
| 0 | "本项目的创新点包括：" | "我的作品有这些创新的地方：" |
| 1 | "与现有方案相比，本项目在以下方面实现了改进：" | "和现在市面上的产品比起来，我的装置有这些不同：" |
| 2 | "本项目在以下几个方面具有创新性：" | "我觉得我的设计有几个和别人不一样的地方：" |

**6. 小学低年级专用过渡**

| 套号 | 表达 |
|------|------|
| 0 | "有一天，我发现……" / "做好以后，我试了试……" / "通过这个项目，我学到了……" |
| 1 | "我注意到……" / "我请同学们帮我试了试……" / "做完这个东西，我觉得……" |
| 2 | "在日常生活中，我经常遇到……" / "我让爸爸妈妈也来试一试……" / "回想整个过程，我最大的收获是……" |

### 段落起始结构多样化

除了过渡语，段落的起始方式也应多样化。以下 4 种结构应交替使用：

**直述式：** 直接陈述事实。
> "本系统采用ESP32-S3作为主控芯片。"

**背景先行式：** 先说原因/背景，再给结论。
> "考虑到系统需要同时处理图像识别和电机控制，选用了双核的ESP32-S3。"

**数据先行式：** 先给数据/结果，再展开说明。
> "经过5轮对比测试，方案二的识别率(93%)明显优于方案一(78%)，因此最终采用方案二。"

**问题先行式：** 先描述遇到的问题，再给出解决方案。
> "传感器读数波动较大的问题，通过增加中值滤波算法得到了解决。"

**使用原则：** 同一段落内保持一种结构；相邻段落尽量使用不同结构；同一项目内四种结构均应出现。

## Common Pitfalls to Avoid

### ❌ 摘要过于笼统
```
本项目设计了一个智能装置，效果良好，具有一定的创新性和实用价值。
```
### ✅ 摘要具体有力
```
本项目设计了一套基于UWB测距的便携式视力测量仪，利用DW1000超宽带模块实现±10cm精度的5米距离自动校准，采用4.2寸电子墨水屏显示E字视标（对比度≥85%）。经测试，系统测距成功率达96%，单次测试时间约3分钟，续航超过8小时。主要创新点包括UWB精确测距替代传统卷尺、电子墨水屏高对比度视标显示。
```
