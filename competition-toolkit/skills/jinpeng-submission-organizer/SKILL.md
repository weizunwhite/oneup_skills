# 金鹏科技论坛 提交材料整理器

## Overview

两个核心功能：
1. **完整性检查** — 对照金鹏提交标准，逐项检查哪些材料已有、哪些还缺
2. **自动整理打包** — 将所有最终版材料复制到标准化的提交文件夹中，照片打包成文档

## 金鹏提交标准文件夹结构

**参照 `/Users/zunwei/Desktop/workup/project2025/金鹏提交示例/` 的真实结构：**

```
提交根目录/                                          ← 以学生名+类别+项目名命名
├── 研究报告.docx                                    ← 研究论文
├── 研究方案.docx                                    ← 开题研究方案
├── 附件1 档案袋封页（后附填写样例）.docx               ← 档案袋封页模板
├── {序号}+{类别}+{项目名称}/                         ← 视频子文件夹
│   ├── 介绍视频.mp4                                 ← 可选，找不到就跳过
│   └── 演示视频.mp4                                 ← 可选，找不到就跳过
└── 研究过程材料/                                     ← 过程材料子文件夹
    ├── 工作日志.docx                                ← 研究日志
    ├── 介绍幻灯片.pptx                              ← 金鹏 PPT
    ├── 研究过程记录与照片.docx                        ← 照片打包文档（本 skill 生成）
    └── 原始研究材料.pdf                              ← 原始资料模板/手抄内容（合并为 PDF）
```

### 关键规范

- **根目录命名**: `{学生姓名}+{类别}+{项目名称}`（如 `袁家雪+发明创造类+老旧小区楼梯间智能消防灭火装置`）
- **视频文件夹命名**: `{序号}+{类别}+{项目名称}`（如 `2+发明创造类+老旧小区楼梯间智能消防灭火装置`）
- **文件名统一简洁**: 不带版本号、日期戳、项目名前缀
- **视频可选**: 如果项目中没有 .mp4 文件，视频文件夹可以为空或不创建
- **照片必须打包**: 散落的照片文件不能直接放在文件夹中，必须整合成 Word 或 PPT 文档

## Workflow

### Step 1: 读取项目信息

从 `项目信息表.md` 获取：
- 项目名称
- 学生姓名（用于提交根目录命名）
- 项目类别（发明创造类/科学论文类/创意设计类）
- 项目序号（学校分配的编号，如果信息表中没有则向用户询问）
- 学科分类
- 学生年级

### Step 2: 扫描可用文件

扫描项目目录中的 `输出/` 和 `原始资料/`（或 `原始材料/`），自动识别每个文件的类别。

**文件识别规则：**

| 匹配模式 | 目标材料 | 提交文件名 |
|----------|---------|-----------|
| `*研究报告*` 或 `*论文*`（.docx） | 研究报告 | `研究报告.docx` |
| `*研究方案*`（.docx） | 研究方案 | `研究方案.docx` |
| `*研究日志*` 或 `*工作日志*`（.docx） | 工作日志 | `工作日志.docx` |
| `*金鹏*PPT*` 或 `*介绍幻灯片*`（.pptx） | 介绍幻灯片 | `介绍幻灯片.pptx` |
| `*原始资料模板*` 或 `*手抄内容*`（.docx） | 原始研究材料 | 合并为 `原始研究材料.pdf` |
| `*档案袋*`（.docx） | 档案袋封页 | `附件1 档案袋封页（后附填写样例）.docx` |
| `手写记录*`（.pdf） | 原始研究材料 | 纳入 `原始研究材料.pdf` |
| `.mp4` 文件 | 视频 | `介绍视频.mp4` 或 `演示视频.mp4` |
| `.jpg/.jpeg/.png` 文件（在 photos_resized/ 或 原始资料/photos/） | 过程照片 | 打包进 `研究过程记录与照片.docx` |

**扫描路径优先级：**
1. `输出/` — 生成的最终版文档
2. `原始资料/` 或 `原始材料/` — 原始材料（手写记录 PDF、照片等）
3. 项目根目录 — 可能有直接放在根目录的文件
4. `提交/` — 已有的旧提交（用于参考，不重复复制）

### Step 3: 完整性检查

逐项对照清单，输出检查结果：

```
🔍 金鹏科技论坛 提交材料完整性检查

📋 项目：{项目名称}
   学生：{姓名}  类别：{类别}  序号：{序号}

━━━━━━━━━━━━━━━━━━━━━

📄 根目录文件：
  ✅ 研究报告.docx      ← {源文件名}
  ✅ 研究方案.docx      ← {源文件名}
  ⬜ 档案袋封页         ← 未找到（需要用户提供模板）

📹 视频文件夹 ({序号}+{类别}+{项目名}/)：
  ✅ 演示视频.mp4       ← {源文件名}
  ⬜ 介绍视频.mp4       ← 未找到（跳过）

📁 研究过程材料/：
  ✅ 工作日志.docx           ← {源文件名}
  ✅ 介绍幻灯片.pptx         ← {源文件名}
  ✅ 研究过程记录与照片.docx   ← 将从 {N} 张照片生成
  ✅ 原始研究材料.pdf         ← {源文件名}

📊 结果：✅ {N}/M 项已就绪，⬜ {K} 项缺失/跳过
```

### Step 4: 照片打包成文档

**这是本 skill 的核心功能之一。** 将项目的所有过程照片整合成一份 Word 文档 `研究过程记录与照片.docx`。

**照片来源（按优先级）：**
1. `photos_resized/` — 已处理的照片
2. `原始资料/photos/` 或 `原始材料/项目照片*/` — 原始照片
3. `原始资料/` 下的散落图片文件

**文档结构：**
```
研究过程记录与照片

一、调研阶段
  [照片1] 说明文字
  [照片2] 说明文字

二、设计阶段
  [照片3] 说明文字
  ...

三、制作阶段
  ...

四、测试阶段
  ...

五、成品展示
  ...
```

**照片分类逻辑：**
根据文件名前缀自动分类（如 `survey_*` → 调研、`design_*` → 设计、`make_*` → 制作、`test_*` → 测试、`product_*` → 成品）。如果文件名无前缀规律，按文件修改时间排序，均匀分配到各阶段。

**生成脚本要点：**
```javascript
// 使用 docx + sharp 生成
// 每张照片：
// - 用 sharp 缩放到宽度 450px（适合 A4 打印）
// - 图片居中显示
// - 下方加灰色说明文字（从文件名推断或通用描述）
// - 照片间距适当，每页放 2-3 张
// 不加页眉页脚（这是附件文档，不是正式论文）
```

### Step 5: 原始研究材料合并

如果有多个原始资料文件（模板 .docx + 手抄内容 .docx + 手写记录 .pdf），需要合并为一个 `原始研究材料.pdf`。

**合并策略：**
- 如果只有一个 PDF（如 `手写记录.pdf`），直接复制并重命名
- 如果有 .docx 原始资料模板，提醒用户打印后手填，再扫描成 PDF
- 如果既有 PDF 又有 .docx，将 .docx 的内容说明和 PDF 路径一并告知用户

### Step 6: 执行整理

创建提交文件夹并复制文件：

```bash
# 提交根目录
BASE="项目目录/提交/{学生姓名}+{类别}+{项目名称}"
mkdir -p "$BASE"
mkdir -p "$BASE/{序号}+{类别}+{项目名称}"
mkdir -p "$BASE/研究过程材料"

# 复制根目录文件
cp "输出/xxx_研究报告.docx"    "$BASE/研究报告.docx"
cp "输出/xxx_研究方案.docx"    "$BASE/研究方案.docx"    # 如果有
cp "档案袋封页模板.docx"       "$BASE/附件1 档案袋封页（后附填写样例）.docx"

# 复制视频（如果有）
cp "原始资料/演示视频.mp4"     "$BASE/{序号}+{类别}+{项目名称}/演示视频.mp4"
# 介绍视频如果没有就跳过，不报错

# 复制研究过程材料
cp "输出/xxx_研究日志.docx"    "$BASE/研究过程材料/工作日志.docx"
cp "输出/xxx_金鹏PPT.pptx"    "$BASE/研究过程材料/介绍幻灯片.pptx"
# 生成的照片文档
cp "照片打包.docx"             "$BASE/研究过程材料/研究过程记录与照片.docx"
# 原始研究材料
cp "手写记录.pdf"              "$BASE/研究过程材料/原始研究材料.pdf"
```

## Notes

1. **不修改源文件** — 只复制，不改动原文件内容
2. **可重复运行** — 补充材料后可以再次运行，会覆盖之前的提交文件夹
3. **视频可选** — 如果找不到 .mp4 文件，跳过视频文件夹，不报错
4. **照片必须打包** — 散落的照片文件不能直接放在提交文件夹中，必须整合成 `研究过程记录与照片.docx`
5. **档案袋封页** — 优先从项目目录中找已有的，找不到则从 `/Users/zunwei/Desktop/workup/project2025/金鹏提交示例/附件1 档案袋封页（后附填写样例）.docx` 复制模板
6. **研究方案** — 有些项目可能没有单独生成研究方案，可以和研究报告共用同一份，或提醒用户补充
