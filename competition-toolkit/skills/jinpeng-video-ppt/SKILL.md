# 金鹏科技论坛 视频PPT + 脚本生成器

## Overview

专门为**北京金鹏科技论坛**视频录制生成两份配套材料：
1. **演示PPT**（.pptx）— 10-13页，用于视频录制时投影辅助
2. **视频脚本**（.docx）— 逐页对应PPT的完整口述稿，学生直接背诵

**核心原则：**
- **6 大板块不变**，严格对应金鹏 6 项必讲要求
- **每个板块可以用 1-3 页展开**，避免单页内容过多看不清
- **PPT备注 = 视频脚本，完全一致，一字不差**
- 每页都有备注，脚本也逐页对应

**IMPORTANT**: Before generating, you MUST also read:
- `/mnt/skills/public/pptx/SKILL.md` and its `pptxgenjs.md` reference for PPT technical implementation
- `/mnt/skills/public/docx/SKILL.md` for Word document generation

## 金鹏视频要求（必须严格遵守）

### 视频内容要求（6个必讲内容）
1. **选题缘由** — 为什么做这个项目
2. **研究设计和方法** — 怎么设计的、用了什么方法
3. **主要过程及结果** — 做了什么、得到什么结果
4. **研究价值** — 这个研究有什么意义和价值
5. **本人做出的主要贡献及参与度** — 自己做了什么、参与了多少
6. **集体项目中每个成员的分工和工作任务**（集体项目必讲）/ 个人项目可改为"感谢与展望"

### 视频拍摄规范
- 时长 ≤ 5 分钟
- 学生全身出镜
- 固定机位，一镜到底，不可剪辑
- mp4 格式，≤ 300MB
- 文件夹命名：序号+项目类别+项目名称

### ⚠️ 匿名化要求（红线！违反即取消资格）
- **不得出现**：学校名称、作者姓名、辅导教师姓名
- PPT 封面、内容页、备注中一律不得包含任何可识别个人信息
- 脚本中也不得出现任何个人信息

## Workflow

### Step 1: Gather Project Information

**Required:**
- 项目名称
- 学生年级（小学/初中）
- 项目类型：个人项目 / 集体项目
- 项目简述
- 竞赛论文或技术文档（强烈建议提供）

**如果提供了论文/技术文档**，直接从中提取所有内容，不需要再问学生。解析论文中的：选题背景、研究方法、实验过程、测试结果、创新点、个人贡献等。

**集体项目额外信息：**
- 成员人数
- 每个成员的分工说明

### Step 2: Select Visual Theme

每个项目必须视觉上独特。参考 `references/design-themes.md` 选择主题。

**主题选择规则：**
1. 根据项目领域匹配主题（消防→暖色、环保→绿色、医疗→蓝色、科技→紫色等）
2. 如果领域不明确，用项目名称首字 Unicode 值取模选择
3. 绝不默认使用同一个主题

### Step 3: Design Slide Structure (10-13 Slides)

**6 大板块固定不变，但每个板块可以用多页展开。** 内容多的板块用 2-3 页，内容少的用 1 页。每一页都有独立的备注和脚本段落。

**推荐 11 页基准结构：**

| 页码 | 内容 | 对应金鹏板块 | 建议时长 | 类型 |
|------|------|-------------|---------|------|
| 1 | 封面（仅项目名称） | — | ~15秒 | cover |
| 2 | 选题缘由：个人故事 | ① 选题缘由 | ~25秒 | story |
| 3 | 选题缘由：背景调研 | ① 选题缘由 | ~25秒 | data |
| 4 | 研究设计：整体方案 | ② 研究设计和方法 | ~25秒 | architecture |
| 5 | 研究设计：核心组件/技术 | ② 研究设计和方法 | ~25秒 | detail |
| 6 | 过程及结果：制作过程 | ③ 主要过程及结果 | ~25秒 | process |
| 7 | 过程及结果：困难与解决 | ③ 主要过程及结果 | ~20秒 | comparison |
| 8 | 过程及结果：测试与数据 | ③ 主要过程及结果 | ~25秒 | data |
| 9 | 研究价值 | ④ 研究价值 | ~30秒 | value |
| 10 | 本人贡献及参与度 | ⑤ 本人贡献及参与度 | ~30秒 | contribution |
| 11 | 成员分工/感谢 | ⑥ 成员分工 | ~15秒 | closing |

**总时长：约 260 秒（4分20秒），留约 40 秒缓冲用于自然停顿和翻页。**

**弹性调整规则：**

| 金鹏板块 | 最少页数 | 最多页数 | 可展开的内容 |
|----------|---------|---------|-------------|
| 封面 | 1 | 1 | 固定1页 |
| ① 选题缘由 | 1 | 2 | 背景调研可单独一页 |
| ② 研究设计和方法 | 1 | 3 | 系统架构、核心组件A、核心组件B |
| ③ 主要过程及结果 | 2 | 3 | 制作过程、困难解决、测试结果 |
| ④ 研究价值 | 1 | 1 | 通常1页足够 |
| ⑤ 本人贡献及参与度 | 1 | 1 | 通常1页足够 |
| ⑥ 成员分工/感谢 | 1 | 1 | 结尾固定1页 |

- **简单项目（10页）：** 选题缘由合并为1页，过程及结果中困难与解决合并到制作过程页
- **复杂项目（13页）：** 研究设计拆为3页（方案+组件A+组件B），过程及结果3页全展开

**关键原则：**
1. 每一页内容不要太挤，宁可多翻一页也不要堆在一起看不清
2. 每一页的讲解时间可以不同（15秒-30秒都正常）
3. 学生只要记住到哪个内容点就翻页即可
4. **每一页都必须有 Speaker Notes，没有例外**

参考 `references/slide-content-guide.md` 获取每页的详细内容指南。

### Step 4: Layout Variant Selection

参考 `references/layout-variants.md` 为每页选择不同布局。

**规则：**
1. 封面：只显示项目名称 + 年级标签，**绝对没有姓名和学校**
2. 相邻页面的布局风格不能重复
3. 同一板块内的多页也要用不同的布局类型（比如"研究设计"的2页不能都用流程图）
4. 每个项目的布局组合要不同
5. 至少使用 3 种以上不同的布局模式

### Step 5: Write Script Content (核心步骤)

**先写完整脚本，再做PPT。** 这样确保内容逻辑通顺。

#### 脚本写作规则：

**语言风格按年级调整：**
- 小学（1-6年级）：口语化，像讲故事，用"我发现...我想到...我做了..."
- 初中（7-9年级）：稍正式，可用技术术语，但首次出现要解释

**每页脚本必须包含：**
- 对应页码标注：`【第X页：标题】`
- 完整口述内容（学生可直接背诵）
- 时间提示：`（约XX秒）`
- 适当的停顿标记：`（停顿1秒）`
- 翻页提示：段末标注 `→ 翻页`（最后一页除外）

**脚本总字数控制：**
- 小学生语速：约 180-200 字/分钟 → 总字数 900-1000 字
- 初中生语速：约 200-220 字/分钟 → 总字数 1000-1100 字
- 注意：页数变多不代表总字数变多！总时长仍然是5分钟，只是分配到更多页上

**匿名化检查：**
脚本中绝对不能出现：
- ❌ "我是XX学校的..."
- ❌ "我叫XXX..."
- ❌ "我的指导老师XXX..."
- ✅ "大家好！我是一名X年级的学生..."（正确开头方式）

#### 各板块脚本内容指南：

**板块0：封面（1页，~15秒，约50字）**
```
【第1页：封面】
大家好！我是一名{年级}的学生。今天给大家介绍我的科技创新项目——{项目名称}。（停顿1秒）
→ 翻页
（约15秒）
```

**板块①：选题缘由（1-2页，共~45秒，约150字）**

如果1页：
```
【第2页：选题缘由】
{个人故事 + 背景调研 + 决定行动}
→ 翻页
（约45秒）
```

如果2页：
```
【第2页：选题缘由 — 我的发现】
{个人观察、真实经历、发现的问题}
→ 翻页
（约25秒）

【第3页：选题缘由 — 背景调研】
{调查数据、现有方案不足、确认问题值得研究}
→ 翻页
（约20秒）
```

**板块②：研究设计和方法（1-3页，共~50秒，约170字）**

如果1页：系统概述 + 关键技术
如果2页：整体方案（架构图）+ 核心组件/技术详情
如果3页：整体方案 + 核心组件A + 核心组件B

每页示例：
```
【第X页：研究设计 — 整体方案】
为了解决这个问题，我设计了{系统名称}。它由{N}个部分组成...
→ 翻页
（约25秒）

【第X+1页：研究设计 — 核心技术】
在关键技术上，我选择了{技术}来实现{功能}，因为{原因}...
→ 翻页
（约25秒）
```

**板块③：主要过程及结果（2-3页，共~65秒，约210字）**

推荐拆法：
- 页A：制作过程（分2-3阶段简述）
- 页B：困难与解决方案（1-2个代表性困难）
- 页C：测试数据与结果（具体数据、成功率等）

如果2页：困难与解决合并到制作过程页

每页示例：
```
【第X页：制作过程】
项目的制作分为{N}个阶段。第一阶段...第二阶段...第三阶段...
→ 翻页
（约25秒）

【第X+1页：困难与解决】
在制作过程中，我遇到了{困难}的问题。经过反复尝试，我发现{解决方案}...
→ 翻页
（约20秒）

【第X+2页：测试与结果】
经过测试，{指标1}达到了{数值}，{指标2}为{数值}...
→ 翻页
（约25秒）
```

**板块④：研究价值（1页，~30秒，约100字）**
```
【第X页：研究价值】
这个项目的价值在于：首先，{实际应用价值}；其次，{社会意义}；未来还可以{发展方向}...
→ 翻页
（约30秒）
```

**板块⑤：本人贡献及参与度（1页，~30秒，约100字）**
```
【第X页：本人贡献及参与度】
在这个项目中，我独立完成了{具体贡献}...整个项目从选题到测试，我的参与度为{程度说明}...
→ 翻页
（约30秒）
```

**板块⑥：成员分工/感谢（1页，~15秒，约50字）**

集体项目：
```
【第X页：成员分工】
我们团队共{N}位成员：成员一负责{任务}，成员二负责{任务}，我主要负责{任务}。
```

### Step 7: Generate PPT

使用 Node.js + pptxgenjs 生成 .pptx 文件。

**代码规则：**

1. **react-icons 渲染图标** — 通过 sharp 转 PNG 后嵌入
2. **每页都有 Speaker Notes** — 内容与脚本文档对应页完全一致
3. **字体策略**：一个标题字体 + 一个正文字体，从主题中选取
4. **阴影辅助函数**：创建 `mkShadow()` 复用
5. **图片安全**：`fs.existsSync()` 检查后再添加
6. **颜色常量**：顶部定义 `C` 对象
7. **输出路径**：`/home/claude/jinpeng_ppt.pptx`

**封面页特殊处理：**
```javascript
// ⚠️ 金鹏匿名要求：封面只有项目名称和年级
// 绝对不能有：姓名、学校名、教师名
slide.addText(projectName, { ... }); // 项目名称
slide.addText(gradeLabel, { ... });  // 如 "小学五年级"
// NO: slide.addText("[姓名]", ...) ← 禁止！
// NO: slide.addText("[学校名称]", ...) ← 禁止！
```

**备注格式（每页都有，与脚本完全一致）：**
```javascript
// 第1页备注
slide1.addNotes(`【第1页：封面】\n\n大家好！...\n\n→ 翻页\n（约15秒）`);

// 第2页备注
slide2.addNotes(`【第2页：选题缘由 — 我的发现】\n\n有一次...\n\n→ 翻页\n（约25秒）`);

// 第3页备注
slide3.addNotes(`【第3页：选题缘由 — 背景调研】\n\n为了了解更多...\n\n→ 翻页\n（约20秒）`);

// ... 以此类推，每页都有对应备注
```
