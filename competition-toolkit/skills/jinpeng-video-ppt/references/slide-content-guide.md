# Slide Content & Script Guide (Jinpeng Multi-Page Version)

## Anonymization Rules (ALL Slides)

**NEVER include anywhere in PPT or script:**
- Student name → use "我"
- School name → omit entirely, or say "我的学校附近" if location context needed
- Teacher name → use "指导老师" or "老师"
- Classmate names → use "同学" "成员一" "队友"
- School uniform logos in photos → warn user to check

---

## Cover: 封面 (1 page, ~15 seconds)

**Jinpeng Area:** None (opening)

**PPT Content:**
- Project full name (large, 36-44pt)
- Subtitle/tagline describing core approach (16-18pt, accent color)
- Grade label: "小学X年级" or "初中X年级" (small, muted)
- Theme icon
- Product hero photo (if available)

**EXCLUDED from cover:**
- ~~Student name~~ ❌
- ~~School name~~ ❌
- ~~Teacher name~~ ❌

**Script (~50 chars):**
```
大家好！我是一名{年级}的学生。今天给大家介绍我的科技创新项目——{项目名称}。（停顿1秒）
→ 翻页
（约15秒）
```

## 板块① 选题缘由 (1-2 pages, total ~45 seconds)

### Option A: 1 page (simple projects)

**Page: 选题缘由 (~45s, ~150 chars)**

PPT Content: 3-part story cards (发现 → 思考 → 行动) or photo + narrative layout

Script Template (Elementary):
```
【第2页：选题缘由】
有一次{personal event or observation}。
我当时就想：{question that formed}？（停顿1秒）
经过调查我发现，{supporting fact or data}。
于是我决定{action taken}，试试能不能解决这个问题。
→ 翻页
（约45秒）
```

### Option B: 2 pages (recommended for most projects)

**Page B-1: 选题缘由 — 我的发现 (~25s, ~80 chars)**

PPT Content: Personal story with photo — the observation, the "aha moment"

Script Template (Elementary):
```
【第2页：选题缘由 — 我的发现】
有一次{personal event}。
我看到{specific observation}，当时就觉得{emotional reaction}。
我想：{question that formed in mind}？
→ 翻页
（约25秒）
```

**Page B-2: 选题缘由 — 背景调研 (~20s, ~70 chars)**

PPT Content: Data cards or stats from investigation — field research, survey results, existing solution analysis

Script Template (Elementary):
```
【第3页：选题缘由 — 背景调研】
为了了解更多，我{research action: 查阅资料/做了问卷/实地调查}。
我发现{key finding with data}，而且{existing solutions' limitation}。
这更坚定了我做这个项目的决心。
→ 翻页
（约20秒）
```

**Key principles for 选题缘由：**
- Story must feel PERSONAL and GENUINE — avoid generic openings like "随着科技的发展..."
- Use specific details: time, place, who was involved, what exactly happened
- Elementary: "我看到了...我想到了...我决定..."
- Middle school: More analytical, can reference broader context

## 板块② 研究设计和方法 (1-3 pages, total ~50 seconds)

### Option A: 1 page (very simple projects)

**Page: 研究设计和方法 (~50s, ~170 chars)**

PPT Content: System diagram + key technology list on one slide

### Option B: 2 pages (recommended)

**Page B-1: 研究设计 — 整体方案 (~25s, ~85 chars)**

PPT Content: System architecture diagram — flow chart, center hub, or layered stack showing major components and their connections

Script Template (Elementary):
```
【第X页：研究设计 — 整体方案】
为了解决这个问题，我设计了{system name}。
它主要分为{N}个部分：{part 1}负责{function}，{part 2}负责{function}，{part 3}负责{function}。
它们通过{connection method}连接在一起工作。
→ 翻页
（约25秒）
```

Script Template (Middle School):
```
【第X页：研究设计 — 整体方案】
基于前期调研，我设计了{system name}的整体方案。
系统采用{architecture description}，主要包含{components}。
{component A}负责{function}，通过{protocol}将数据传给{component B}进行处理，最终由{component C}{output action}。
→ 翻页
（约25秒）
```

**Page B-2: 研究设计 — 核心技术/组件 (~25s, ~85 chars)**

PPT Content: Key component photos + specs, or technology choice with rationale. Use Content-B (photo + list) or Content-A (card grid)

Script Template (Elementary):
```
【第X页：研究设计 — 核心技术】
在这个项目中，我用了{key technology/component}。
我选择它是因为{reason in simple terms}。
{optional: brief explanation of how it works in everyday language}
→ 翻页
（约25秒）
```

### Option C: 3 pages (complex hardware projects)

**Page C-1:** 整体方案 (same as B-1)
**Page C-2:** 核心组件A — e.g., sensing module, hardware details
**Page C-3:** 核心组件B — e.g., control logic, software/algorithm

**Key principles for 研究设计和方法：**
- Emphasize DESIGN THINKING, not just a parts list
- "I chose X because Y" beats "I used X"
- Architecture diagram is essential — even if simplified for elementary students
- For hardware: show input → processing → output flow
- For software: show algorithm or data flow
- Each page in this section should have a DIFFERENT visual layout

## 板块③ 主要过程及结果 (2-3 pages, total ~65 seconds)

This is the LONGEST section and carries the most content weight.

### Option A: 2 pages

**Page A-1: 制作过程 + 困难解决 (~35s, ~120 chars)**

PPT Content: Timeline/process layout (3 stages) + one highlighted difficulty-solution pair

**Page A-2: 测试与结果 (~30s, ~100 chars)**

PPT Content: Hero number stats (3-4 key metrics) + test photo

### Option B: 3 pages (recommended)

**Page B-1: 制作过程 (~25s, ~80 chars)**

PPT Content: Content-D (timeline/process) — 3 columns showing development stages with photos

Script Template (Elementary):
```
【第X页：制作过程】
项目的制作分为三个阶段。
第一步，我{phase 1 action}；
第二步，我{phase 2 action}；
第三步，我{phase 3 action}。
→ 翻页
（约25秒）
```

Script Template (Middle School):
```
【第X页：制作过程】
项目实施分为三个阶段：
第一阶段是{phase name}，主要完成了{work}；
第二阶段是{phase name}，重点解决了{work}；
第三阶段是{phase name}，进行了{work}。
→ 翻页
（约25秒）
```

**Page B-2: 困难与解决 (~20s, ~65 chars)**

PPT Content: Content-C (comparison arrows) — 1-2 problem→solution pairs with color coding (warm for problems, cool for solutions)

Script Template (Elementary):
```
【第X页：困难与解决】
在制作过程中，我遇到了一个难题：{specific difficulty}。
一开始我试了{failed attempt}，但不行。后来我想到{solution approach}，终于解决了。
→ 翻页
（约20秒）
```

Script Template (Middle School):
```
【第X页：困难与解决】
在调试过程中，遇到了{difficulty}的问题。
经过分析，我发现原因是{root cause}，于是采用{solution approach}的方法，成功解决了这个问题。
→ 翻页
（约20秒）
```

**Page B-3: 测试与结果 (~25s, ~80 chars)**

PPT Content: Content-E (hero numbers) — 3-4 big stat cards at top + test photo + brief description

Script Template (Elementary):
```
【第X页：测试与结果】
经过反复测试，结果显示：{metric 1}达到了{value}，{metric 2}为{value}。
比如{specific real-world example of it working}。
这说明我的项目确实能{solve the problem}。
→ 翻页
（约25秒）
```

Script Template (Middle School):
```
【第X页：测试与结果】
为了验证方案的可行性，我进行了{test method}。
测试数据表明：{metric 1}达到{value}，{metric 2}为{value}，{metric 3}控制在{value}以内。
这些结果证明系统{conclusion}。
→ 翻页
（约25秒）
```

**Key principles for 主要过程及结果：**
1. MUST include specific numbers/data — vague results like "效果很好" are not acceptable
2. Include at least ONE difficulty + solution to show problem-solving ability
3. Process should show hands-on work evidence
4. Most impressive difficulty-solution should get the most detail
5. This section should contain at least 1 photo (process or testing)

## 板块④ 研究价值 (1 page, ~30 seconds)

**Page: 研究价值 (~30s, ~100 chars)**

PPT Content: 3 value dimensions as cards or icon rows:
1. 实际应用价值 — Who benefits? How? In what scenarios?
2. 社会意义 — What broader problem does this address?
3. 推广/发展前景 — Could this be scaled up? Future potential?

Script Template (Elementary):
```
【第X页：研究价值】
我觉得这个项目很有价值。
首先，它可以帮助{who}解决{what problem}。
而且，如果更多人用上这样的装置，{broader benefit}。
未来，我还想{improvement plan}，让它帮助更多的人。
→ 翻页
（约30秒）
```

Script Template (Middle School):
```
【第X页：研究价值】
本项目具有以下研究价值：
在实际应用方面，{practical value and beneficiaries}。
从社会意义来看，{connection to broader issues}。
在推广前景方面，{scalability and future development}。
→ 翻页
（约30秒）
```

**Key principles for 研究价值：**
- Go BEYOND the project itself — connect to real-world impact
- Mention specific beneficiary groups
- Be honest about scope — small projects can have big implications
- NEVER cut this section — uniquely important for Jinpeng evaluation

## 板块⑤ 本人贡献及参与度 (1 page, ~30 seconds)

**Page: 本人贡献及参与度 (~30s, ~100 chars)**

PPT Content: Card grid (2×3) showing 4-6 specific contributions across categories:
1. 调研 (Research/Investigation)
2. 创意设计 (Creative design decisions)
3. 动手制作 (Physical construction)
4. 编程 (Programming, if applicable)
5. 测试迭代 (Testing and iteration)
6. 自学新技能 (Self-learning)

Bottom: Participation level statement + advisor disclaimer

Script Template (Elementary):
```
【第X页：本人贡献及参与度】
在这个项目中，我自己完成了很多工作。
我亲自{contribution 1}，自己动手{contribution 2}，还学会了{new skill}。
遇到不懂的，我会先自己查资料尝试，实在不行才请教老师。
整个项目的核心工作都是我独立完成的。
→ 翻页
（约30秒）
```

Script Template (Middle School):
```
【第X页：本人贡献及参与度】
关于我的个人贡献：
我独立完成了{research work}、{design work}、{construction work}和{testing work}。
在过程中，我自学了{new skill}。
指导老师提供了方向性建议，但所有核心设计、编程、制作和调试均由我独立完成，参与度为{percentage or description}。
→ 翻页
（约30秒）
```

**Key principles for 本人贡献及参与度：**
- BE SPECIFIC — "我自己焊接了电路板" beats "我负责硬件部分"
- Participation level must be EXPLICIT
- Advisor acknowledgment brief and subordinate
- NEVER cut this section — uniquely important for Jinpeng evaluation

## 板块⑥ 成员分工 / 感谢 (1 page, ~15 seconds)

### For Group Projects:

**Page: 成员分工 (~15s, ~50 chars)**

PPT Content: One card per team member (use "成员一/二/三", NOT real names) + "谢谢大家！"

Script Template:
```
【第X页：成员分工】
我们团队共{N}位成员：
成员一主要负责{tasks}；成员二负责{tasks}；我主要负责{your tasks}。
以上就是我的项目介绍，感谢各位评委老师！
（约15秒）
```

### For Individual Projects:

**Page: 感谢 (~15s, ~30 chars)**

PPT Content: "感谢聆听" (large) + project name (accent) + "欢迎评委老师提问！" — NO student name, NO school

Script Template:
```
【第X页：感谢】
以上就是我的项目介绍。感谢各位评委老师的聆听！欢迎提问。
（约15秒）
```

## Grade-Level Language Guide

### Elementary School (小学 1-6年级)
- 口语化：像跟朋友讲故事一样
- 多用"我"开头：我发现、我想到、我学会了、我觉得
- 句子控制在15字以内
- 允许表达情感：害怕、兴奋、自豪、好奇
- 用日常语言解释技术："传感器就像机器的'眼睛'"
- 节奏明快，避免冗长

### Middle School (初中 7-9年级)
- 可以使用技术术语，但首次出现简要解释
- 更注重逻辑推理过程
- 数据描述更精确
- 仍保持第一人称
- 可使用连接词增强逻辑性："基于此""因此""进一步"
- 语气沉稳但不刻板
