## 重要前提：模块级开发

本技能体系中所有传感器、通信模块、显示模块、执行器均为**集成模块**（非裸元器件）。
模块内部已集成必要的电阻、电容、稳压、电平转换等电路。

接线说明一律基于模块排针（VCC/GND/DATA/SCL/SDA等），不涉及裸芯片引脚。
不要在生成的代码注释或接线说明中提示"需外接上拉电阻""需分压电路"等，除非 sensor-catalog.json 的 notes 中明确标注（如HC-SR04的Echo引脚保护）。

## 参考资料优先原则

在生成任何模块的驱动代码之前，**必须先检查** `references/` 目录下是否存在该模块的参考资料：

```
~/.claude/skills/hardware-common/references/{模块名}/
```

如果存在，按以下优先级使用：
1. **notes.md** — 开发者的实战笔记和踩坑记录，最高优先级，必须遵守其中的建议
2. **官方例程**（.ino / .cpp） — 已验证可用的代码，生成新代码时优先参考其初始化流程和调用方式
3. **数据手册**（.pdf） — 补充 sensor-catalog.json 中没有的细节

如果 references/ 下没有对应模块的资料，则按 sensor-catalog.json 的信息正常生成。
如果 notes.md 中的信息与 sensor-catalog.json 冲突，以 notes.md 为准（因为是实际验证过的）。

## Skills 自检与纠错机制

**Skills 文件（包括本 SKILL.md、sensor-catalog.json、board-profiles.json）可能存在错误。**
不要盲目信任，遵守以下规则：

### 优先级（从高到低）
1. **用户当前对话中的明确指示** — 最高优先级，永远覆盖 Skills 内容
2. **references/ 中的实战笔记和验证过的例程** — 来自实际项目验证
3. **你自身的知识** — 如果 Skills 数据与你确信的事实矛盾，以你的知识为准
4. **sensor-catalog.json / board-profiles.json** — 作为参考，不作为绝对真理

### 发现问题时
- 如果 catalog 中的引脚定义、库名、协议等与你的知识明显矛盾，**直接告诉用户**，说明矛盾点，按正确的方式生成代码
- 如果生成的代码编译失败或运行异常，优先考虑是否是 Skills 数据有误，而不是反复用同样的方式重试
- 如果用户纠正了某个模块的信息，提醒用户更新 sensor-catalog.json 或 references/ 中的 notes.md

### 不要做的事
- 不要因为 catalog 里写了某个库名就强行使用，如果你知道有更好或更新的库，直接说明
- 不要因为 catalog 里的默认引脚就忽略用户项目的实际接线
- 不要把 Skills 文件的格式规范看得比代码能跑更重要

# Hardware PlatformIO Project Generator

## Overview

Generate production-ready PlatformIO project structures for hardware projects. This generator:
1. Creates proper directory structure with `src/`, `lib/`, `include/`
2. Generates `platformio.ini` with correct board, framework, and library dependencies
3. Copies required sensor drivers into `lib/` as local libraries
4. Creates `pins.h` with validated pin assignments
5. Creates `config.h` for project parameters
6. Generates `main.cpp` with initialization, loop structure, and sensor integration

## Data Files

Read these BEFORE generating any project:

```
Shared data:
/mnt/skills/user/hardware-common/
├── sensor-catalog.json    ← Sensor specs, libraries, default pins
├── board-profiles.json    ← Board configs, pin constraints
```

## Project Structure Template

```
project-name/
├── platformio.ini          ← Board config, library deps, build flags
├── include/
│   ├── pins.h              ← All pin definitions (centralized)
│   └── config.h            ← Project parameters (thresholds, intervals, etc.)
├── src/
│   └── main.cpp            ← Main program
├── lib/
│   ├── DHT22Driver/        ← Local sensor drivers (from sensor library)
│   │   ├── DHT22Driver.h
│   │   └── DHT22Driver.cpp
│   └── README              ← PlatformIO lib readme
└── README.md               ← Project description, wiring diagram
```

## Workflow

### Step 1: Gather Requirements

Ask the user (or extract from context):
- **Board**: ESP32-S3 / ESP32 DevKit / Arduino Nano
- **Sensors/Modules**: List of sensors needed
- **Features**: WiFi? Bluetooth? MQTT? Display?
- **Project name**: For directory and files

### Step 2: Validate Pin Assignments

1. Look up each sensor's default pins for the target board from `sensor-catalog.json`
2. Check for conflicts using `board-profiles.json`:
   - I2C address conflicts
   - ADC2 + WiFi conflicts (ESP32)
   - SPI bus sharing
   - Boot pin restrictions
3. Reassign pins if conflicts found
4. Generate `pins.h`

### Step 3: Generate platformio.ini

```ini
; PlatformIO Configuration
; Project: {project_name}
; Board: {board_display_name}
; Generated by hardware-platformio-project skill

[env:{env_name}]
platform = {platform}
board = {board_id}
framework = arduino
monitor_speed = 115200
upload_speed = 921600

; Library dependencies
lib_deps =
    {lib_dep_1}
    {lib_dep_2}

; Build flags
build_flags =
    -D PROJECT_NAME="\"{project_name}\""
    -D BOARD_TYPE="\"{board_type}\""
```

**Board mapping:**
| Board | platform | board | env_name |
|-------|----------|-------|----------|
| ESP32 DevKit | espressif32 | esp32dev | esp32dev |
| ESP32-S3 | espressif32 | esp32-s3-devkitc-1 | esp32s3 |
| Arduino Nano | atmelavr | nanoatmega328 | nano |

### Step 4: Generate pins.h

```cpp
#ifndef PINS_H
#define PINS_H

// ============================================
// 引脚配置 - {project_name}
// 开发板: {board_name}
// 生成时间: {date}
// ============================================

// --- I2C 总线 ---
#define PIN_SDA     {sda}
#define PIN_SCL     {scl}

// --- 传感器引脚 ---
#define PIN_DHT     {dht_pin}    // DHT22 数据引脚
#define PIN_TRIG    {trig_pin}   // 超声波 Trig
#define PIN_ECHO    {echo_pin}   // 超声波 Echo

// --- 执行器引脚 ---
#define PIN_SERVO   {servo_pin}  // 舵机信号
#define PIN_BUZZER  {buzzer_pin} // 蜂鸣器

// --- 通信引脚 ---
#define PIN_UART_TX {tx_pin}     // 外设串口 TX
#define PIN_UART_RX {rx_pin}     // 外设串口 RX

#endif
```

### Step 5: Generate config.h

```cpp
#ifndef CONFIG_H
#define CONFIG_H

// ============================================
// 项目配置参数 - {project_name}
// ============================================

// --- 系统参数 ---

#define LOOP_INTERVAL   100     // 主循环间隔(ms)

// --- WiFi 配置 ---
#define WIFI_SSID       "your-wifi"
#define WIFI_PASSWORD   "your-password"

// --- 传感器阈值 ---

#define DIST_MIN        10      // 最小距离(cm)
#define DIST_MAX        200     // 最大距离(cm)

// --- MQTT 配置(如需要) ---
// #define MQTT_SERVER   "bemfa.com"
// #define MQTT_PORT     9501
// #define MQTT_TOPIC    "your-topic"

#endif
```

### Step 6: Generate main.cpp

Follow this structure:
```cpp
#include <Arduino.h>
#include "pins.h"
#include "config.h"
// Include sensor drivers...

// === 全局对象 ===
// Sensor objects...

// === 初始化 ===
void setup() {
    Serial.begin(SERIAL_BAUD);
    Serial.println("[系统] 启动中...");
    
    // 初始化传感器
    // ...
    
    Serial.println("[系统] 初始化完成");
}

// === 主循环 ===
void loop() {
    // 读取传感器
    // 处理逻辑
    // 输出/显示
    
    delay(LOOP_INTERVAL);
}
```

## Key Rules

1. **Library deps** come from `sensor-catalog.json` → `libraries.platformio` field
2. **Never use `delay()` in production** for long waits — use `millis()` timing
3. **ESP32 PWM**: Use `ledcSetup()` + `ledcAttachPin()` + `ledcWrite()`, NOT `analogWrite()`
4. **ESP32 Servo**: Use `ESP32Servo` library, NOT standard `Servo`
5. **Multiple I2C devices**: List all addresses in comments, warn about conflicts
6. **WiFi + ADC**: Only assign analog sensors to ADC1 pins

## Output

Generate all files and copy to `/mnt/user-data/outputs/{project-name}/`
Present the project folder to the user.
